# News Portal API

API для новостного портала с функционалом только на чтение (вывод данных).

## Возможности

- Получение списка новостей с фильтрацией и пагинацией
- Получение конкретной новости
- Подсчёт количества новостей по фильтру
- Получение списка категорий и тегов

## Требования

- Go 1.24+
- PostgreSQL 14+
- Docker (опционально)

## Запуск

### 1. Запуск базы данных

```bash
docker compose up -d
```

### 2. Применение миграций

```bash
go run ./cmd/migration up
```

### 3. Запуск сервера

```bash
go run ./cmd/portal
```

Сервер запустится на `:8080` (настраивается через `HTTP_ADDR` в `.env`).

## Конфигурация

Конфигурация читается из файла `.env` или переменных окружения:

| Переменная     | По умолчанию                                          | Описание          |
|----------------|-------------------------------------------------------|-------------------|
| HTTP_ADDR      | `:8080`                                               | Адрес HTTP сервера|
| DATABASE_URL   | `postgres://test:test@localhost:5432/test?sslmode=disable` | URL подключения к БД |

## Тестирование

### Unit-тесты

```bash
go test ./...
```

### API-тесты

После запуска сервера выполните скрипт для проверки всех эндпоинтов:

```bash
./test_api.sh
```

Скрипт проверяет:
- GET /v1/getCategories — получение категорий
- GET /v1/getTags — получение тегов
- GET /v1/countNews — подсчёт новостей (с фильтрами и без)
- GET /v1/listNews — список новостей (с фильтрами и пагинацией)
- GET /v1/getNews — конкретная новость (включая ошибки 400, 404)

## API Endpoints

| Метод | Endpoint         | Описание                           |
|-------|------------------|------------------------------------|
| GET   | /v1/listNews     | Список новостей (без content)      |
| GET   | /v1/countNews    | Количество новостей по фильтру     |
| GET   | /v1/getNews      | Конкретная новость (с content)     |
| GET   | /v1/getCategories| Список категорий                   |
| GET   | /v1/getTags      | Список тегов                       |

Подробная спецификация: [docs/openapi.yaml](docs/openapi.yaml)

## Структура проекта

```
.
├── cmd/
│   ├── portal/main.go      # Точка входа HTTP сервера
│   └── migration/main.go   # Утилита для миграций БД
├── internal/
│   ├── config/             # Загрузка конфигурации
│   ├── domain/             # Доменные модели и интерфейсы
│   │   └── mocks/          # Моки для тестирования
│   ├── endpoints/public/   # HTTP обработчики
│   ├── repository/         # Слой работы с БД
│   ├── service/            # Бизнес-логика
│   │   └── mocks/          # Моки для тестирования
│   └── svcerrs/            # Ошибки сервиса
├── migrations/             # SQL миграции (goose)
├── docs/
│   ├── requirements.md     # Техническое задание
│   ├── openapi.yaml        # OpenAPI спецификация
│   └── schema.sql          # Схема БД (для документации)
├── .env                    # Конфигурация
├── docker-compose.yml      # Docker Compose для БД
└── test_api.sh             # Скрипт тестирования API
```

```

│                    HTTP Handler                          │

└─────────────────────────┬───────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐

└─────────────────────────┬───────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│                    Repository                            │
│                (internal/repository)                     │
```

### Слои

| Слой       | Пакет                    | Ответственность |
|------------|--------------------------|-----------------|
| Handler    | `internal/endpoints/public` | Парсинг HTTP запросов, формирование ответов |
| Service    | `internal/service`       | Валидация, бизнес-логика |
| Repository | `internal/repository`    | Работа с БД |
| Domain     | `internal/domain`        | Модели данных, интерфейсы |

### Миграции

```bash
# Применить все миграции
go run ./cmd/migration up

# Откатить последнюю миграцию
go run ./cmd/migration down

# Откатить все миграции
go run ./cmd/migration reset

# Статус миграций
go run ./cmd/migration status
```
